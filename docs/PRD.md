## 一、产品概述

### 1.1 背景

朋友聚餐吃火锅时，各种食材的涮煮时间不同，容易搞混或者忘记放下去多久了，尤其是人多时更难兼顾。现有解法要么靠记忆，要么土方法掐手机秒表，体验差且分心。

### 1.2 产品定位

**「熟了吗？」** 是一款专为火锅场景设计的涮煮计时微信小程序。帮助用户在涮火锅时精准掌控每种食材的火候，支持多食材并行计时，并提供多人同桌实时同步功能，让全桌都能共享计时看板。

### 1.3 目标用户

- 吃火锅爱好者，在意食材熟度和口感
- 2-8 人小聚，希望全桌共享计时状态
- 注重健康饮食，需要精准控制食物烹饪时间

### 1.4 核心价值

| 价值点 | 说明 |
|-------|------|
| 精准控时 | 参考专业数据（海底捞推荐时间），三档时间选择（安全/推荐/完全煮熟）|
| 智能适配 | 根据锅底温度系数自动调整实际涮煮时间 |
| 多人共享 | WebSocket 实时同步，全桌同一看板 |
| 个性定制 | 用户可自定义食材和锅底，满足特殊饮食需求 |

### 2.3 UI / 交互规范

#### 主题风格
- **主色**：深红 `#8B0000` / 红 `#C0392B`
- **背景**：暖橙 `#FFF5E6`
- **辅助色**：灰 `#7F8C8D`、白 `#FFFFFF`

- **移除所有震动反馈**（保留设置开关但不执行）

## 三、系统架构

```
┌─────────────────────────────────────────┐
│          微信小程序（TypeScript）          │
│  pages: home / foods / room / history   │
│         pot / settings / custom-food    │
│         custom-pot                      │
│  stores: timer-store / room-store       │
│  services: api / websocket / notify     │
└──────────┬──────────────────────────────┘
           │ HTTPS / WSS
┌──────────▼──────────────────────────────┐
│    CloudBase 云托管（Node.js / Fastify）   │
│  /api/foods  /api/pots  /api/stats       │
│  /ws  (WebSocket 多人同步网关)             │
└──────────┬──────────────────────────────┘
           │ CloudBase SDK
┌──────────▼──────────────────────────────┐
│         CloudBase 数据库                  │
│  foods 集合（40+ 条）                     │
│  pots 集合（7 条）                        │
└─────────────────────────────────────────┘
```
